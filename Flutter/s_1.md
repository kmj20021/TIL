## main(){}
앱이 실행될 때 가장 먼저 호출되는 함수   

## runApp()
Flutter 앱의 **루트 위젯**을 화면에 띄우는 함수

## 앱 전체 구조를 담당하는 주요 위젯들 

### MaterialApp()  
머티리얼 디자인 기반 앱의 루트 위젯(앱 시작시)   
가장 많이 사용됨
주요 속성   
- home: 앱이 처음 실행될 때 보여줄 화면   
- routes: 페이지 이동을 위한 경로 설정  
```ruby
void main() {
    runApp(
        MaterialApp(
            home: MainScreen(),
            routes: {'/detail': (context) => DetailScreen(),},
        ),
   );
}
```
#### BuildContext
Flutter에서 모든 위젯은 트리구조로 되어 있다.   
BuildContext는 현재 위젯이 그 트리 안에서 **어디 있는지 나타내는 참조값.**   
Navigator.push(context, ...) : 현재 위치에서 다른 페이지 이동   
Theme.of(context) : 현재 테마 정보 가져오기   
MediaQueery.of(context) : 화면 크기, 방향 등 디바이스 정보 가져오기   
Scaffold.of(context) : 현재 Scaffold에 접근해서 SnackBar 띄우기   


### CupertinoApp()
ios스타일 앱을 위한 루트 위젯(앱 시작시)   
CupertinoPageScaffole, CupertinoNavigationBar와 함께 사용

### WidgetsApp()
MaterialApp과 CupertinoApp의 공동 기반   
테마나 네비게이션 없이 최소한의 **앱 구조 제공**(건물 뼈대)   
커스텀 프레임 워크 만들 때 사용   
```ruby
WidgetsApp(   
    color: Colors.blue,   
    builder: (context, child) => child!,   
)   
```

### Scaffold
각 **페이지의 기본 레이아웃** 구조 제공(안테리어)   
appBar : 상단 앱바(제목, 아이콘 등)      
body : 화면 주요 내용 영역      
floatingActionButtor : 화면 하단의 둥근 버튼   
drawer : 왼쪽에서 나오는 사이드 메뉴   
bottomNavigationBar : 하단 탭 네비게이션 바   
backgroundColor : 배경색 설정   
bottomSheet : 하단에 고정된 시트 형태의 UI   
persistentFooterButtons : 하단에 고정된 버튼들    
resizeToAvoidBottom : 키보드가 올라올 때 화면이 자동으로 밀려 올라가게 할지 여부   

```ruby
Scaffold(
    appBar: AppBar(title: Text("홈")),
    body: Center(child: Text("내용")),
)
```

### SafeArea
노치나 상태바 영역을 피해서 UI를 배치
디바이스마다 다른 화면 영역을 자동으로 고려
```ruby
SafeArea(
    child: Text("안전한 영역에 표시"),
)
```
