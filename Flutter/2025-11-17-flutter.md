## 라우팅이란?
화면 또는 페이지를 이동하는 기능   
이동 시 화면을 바꾸고 필요한 데이터를 전달하면서 화면 이동 이력을 관리함.

SPA(single page application)
한 화면 안에서 Route만 바뀜
여러 Route가 있음

URI : 집을 식별하는 방법 전체(identifier)
URL : 집의 실제 주소(location)
URN : 집의 고유 이름(name)

## 기본 네비게이터(Navvigator.push)
push -> 페이지 위에 새 페이지 올리기
pop  -> 올렸던 페이지 제거

```ruby
Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => const ProfileScreen()),
);
//
Navigator.pop(context);
```

## Named Routes
main.dart를 다음과 같이 설정하는 방법

```ruby
MaterialApp(
  initialRoute: '/',
  routes: {
    '/': (context) => const HomeScreen(),
    '/login': (context) => const LoginScreen(),
  },
);
//
Navigator.pushNamed(context, '/login');
```

## GetX 라우팅
개발자가 편한 방법
url, 파라미터, 애니메이션 모두 지원

```ruby
GetMaterialApp(
  initialRoute: '/',
  getPages: [
    GetPage(name: '/', page: () => const HomeScreen()),
    GetPage(name: '/login', page: () => const LoginScreen()),
  ],
);
// 이동
Get.toNamed('/login');

// 뒤로가기
Get.back();

// 파라미터 전달
Get.toNamed('/detail?id=3');

// 받기
final id = Get.parameters['id'];
```

## 플러터 프로젝트 생성 방법

플러터 프로젝트는 터미널에서 명령어로 생성하거나 IDE에서 템플릿을 선택해 만들 수 있음
가장 기본적인 생성 방식   
- Flutter SDK 설치 후 환경변수 설정   
- 터미널에서 flutter create 프로젝트명 입력   
- 생성된 폴더를 VS Code나 Android Studio로 열어 개발을 시작한다.   
- lib/main.dart가 기본 진입점이며 여기서 앱 구조가 시작된다.   

## Assets의 구성
Assets는 이미지, 폰트, GIF, JSON 파일처럼 앱에서 사용할 **정적 리소스**를 의미한다.   
   
- 프로젝트 루트에 assets/ 폴더를 만든다.   
- 예를 들어 이미지가 필요하다면 assets/images/ 구조로 저장한다.   
- pubspec.yaml에서 아래와 같이 등록한다.   
   
```ruby
assets:
  - assets/images/
```
   
- 등록 후 flutter pub get 을 실행하면 사용 가능하다.

## GetX 라우트 설정
GetX는 상태관리와 라우팅을 간편하게 해주는 라이브러리이다.   
   
- GetMaterialApp으로 앱 엔트리를 감싼다.   
- getPages 속성에 사용할 라우트를 정의한다.   
   
```ruby
getPages: [
  GetPage(name: '/home', page: () => HomePage()),
  GetPage(name: '/login', page: () => LoginPage()),
]
```
   
- 이동은 Get.toNamed('/home'), Get.offNamed('/login') 같은 방식으로 한다.   

## shared_preferences 라이브러리란?

shared_preferences는 간단한 데이터를 로컬 저장소에 보관할 수 있는 플러터 라이브러리다.
앱을 종료해도 데이터가 유지되며 로그인 정보, 설정값, 카운트 값 등 가벼운 데이터를 저장할 때 사용한다.

``` ruby
final prefs = await SharedPreferences.getInstance();
prefs.setInt('counter', 1);
final value = prefs.getInt('counter');
```
